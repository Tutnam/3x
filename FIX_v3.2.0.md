# Решение проблемы "Ошибка при создании профиля" в v3.2.0

## Проблема
После обновления панели 3x-ui с v2.9.4 на v3.2.0 бот перестал работать и выдает ошибку "Ошибка при создании профиля" в Telegram.

## Причины и решения

### ✅ Исправление 1: Удаление параметров Reality из клиентов
**Статус:** ВЫПОЛНЕНО

В v3.2.0 параметры Reality (fingerprint, publicKey, shortId, spiderX) должны находиться только в `streamSettings` инбаунда, а не в самом объекте клиента.

**Что было изменено:**
- Удалены параметры Reality из функций `create_vless_profile()` и `create_static_client()`

### ✅ Исправление 2: Обновление API endpoints
**Статус:** ВЫПОЛНЕНО

Все API endpoints обновлены для использования правильного пути v3.2.0: `/P2wJeRRhxP/panel/api/...`

**Что было изменено:**
- Добавлен метод `_build_url()` для централизованного построения URL
- Обновлены функции: `get_inbound()`, `update_inbound()`, `get_user_stats()`, `get_global_stats()`, `get_online_users()`

### ⚠️ Исправление 3: Авторизация через API Token (ТРЕБУЕТСЯ)
**Статус:** ТРЕБУЕТСЯ ДЕЙСТВИЕ ПОЛЬЗОВАТЕЛЯ

Панель v3.2.0 требует использования API Token для надежной авторизации вместо логина/пароля.

## Что нужно сделать

### 1. Сгенерировать API Token
1. Откройте браузер и перейдите в веб-интерфейс панели
2. Залогиньтесь в панель
3. Перейдите в **Settings** → **Security** → **API Tokens**
4. Нажмите кнопку создания нового токена (например "Bot Token")
5. Скопируйте сгенерированный токен

### 2. Обновить конфигурацию
1. Откройте файл `/src/.env`
2. Добавьте строку:
   ```
   XUI_API_TOKEN=<ваш_api_token>
   ```
   где `<ваш_api_token>` это скопированный токен

3. Сохраните файл

### 3. Перезапустить бота
```bash
python src/app.py
```

## Проверка работы

При запуске бота в логах должно появиться:
```
ℹ️  Using API token for authentication
✅ API token set for authorization
```

Вместо ошибки 404 при логине.

## Если логин все еще не работает

Попробуйте эти варианты:

1. **Проверьте API Token**
   - Убедитесь что скопировали токен полностью
   - Токен не должен содержать пробелов в начале/конце

2. **Проверьте XUI_BASE_PATH**
   - Откройте браузер и перейдите на панель
   - Посмотрите URL: `https://domain.com/PATH_HERE/`
   - `PATH_HERE` это значение XUI_BASE_PATH

3. **Проверьте логи**
   - Посмотрите полный текст ошибки в логах при запуске бота
   - Ошибка должна содержать URL который пытался использовать бот

## Файлы которые были изменены

- `src/functions.py` - API функции обновлены для v3.2.0
- `src/config.py` - добавлена поддержка XUI_API_TOKEN
- `src/.env` - добавлены комментарии об API Token
